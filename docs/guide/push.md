# 推送功能

推送功能是 DooPush 平台的核心，为您提供灵活强大的推送通知能力。支持多种推送类型、精准的目标选择、丰富的内容格式和详细的效果分析。

## 📨 推送概述

DooPush 支持以下推送功能：

- **多种推送类型** - 单推、批量、广播、定时推送
- **精准目标选择** - 设备、标签、分组、平台筛选
- **丰富内容格式** - 标题、内容、角标、自定义载荷
- **消息模板系统** - 可复用的推送模板
- **完整效果分析** - 推送统计和趋势分析

## 🚀 发送推送

### 访问推送发送页面

在左侧导航点击 **"发送推送"** 进入推送发送页面。

### 推送类型

#### 📱 单设备推送 (Single)
向指定的单个设备发送推送通知。

**适用场景**：
- 个性化通知
- 重要消息提醒
- 问题设备测试

**配置要求**：
- **设备 Token** - 目标设备的完整推送 Token
- 从设备管理页面获取或通过 SDK 上报

#### 📋 批量推送 (Batch)
向多个指定设备同时发送推送通知。

**适用场景**：
- 会员专享通知
- 特定用户群体消息
- A/B 测试

**配置要求**：
- **设备 Token 列表** - 多个设备 Token，用逗号分隔
- 最多支持 1000 个设备

**示例格式**：
```
token1,token2,token3
```

#### 🏷️ 标签推送 (Tags)
向拥有特定标签的设备发送推送通知。

**适用场景**：
- 用户分层推送（VIP、普通用户）
- 地域性推送（北京、上海用户）
- 兴趣推送（体育、科技爱好者）

**配置要求**：
- **标签条件** - 至少添加一个标签筛选条件，UI 通过「标签选择器」逐条增删
- 多条标签之间为 **OR 并集**：服务端逐条按 `tag_name=tag_value` 取设备 Token 然后合并去重，不会做 AND 交集

**标签条件示例**（UI 中显示为 `tag_name=tag_value`）：
- `user_level=vip`
- `city=beijing`
- `interest=sports`

::: warning ⚠️ AND 交集请用设备分组
当前后端 `getDeviceTokensByTags` 实现是 OR 取并集。如需「同时满足多个标签」（AND 语义），请改用「设备分组」并在分组规则里组合多条 `equals` / `in` 条件。
:::

#### 📢 广播推送 (Broadcast)
向应用的所有活跃设备发送推送通知。

**适用场景**：
- 重要公告
- 系统维护通知
- 全用户活动

**配置选项**：
- **平台筛选** - 可选择 iOS、Android 或全部平台
- **厂商筛选** - Android 设备可按厂商筛选：
  - Google FCM（通用 Android 设备）
  - 华为 HMS（华为设备）
  - 荣耀推送（荣耀设备）
  - 小米推送（小米/红米设备）
  - OPPO推送（OPPO/OnePlus设备）
  - VIVO推送（VIVO/iQOO设备）
  - 魅族推送（魅族设备）

#### 👥 分组推送 (Groups)
向特定设备分组发送推送通知。

**适用场景**：
- 基于规则的精准推送
- 复杂条件筛选
- 批量目标管理

**配置要求**：
- **设备分组** - 选择预先创建的设备分组
- 支持选择多个分组

::: tip 当前推送目标只覆盖离线设备
后端 `getTargetDevices` 在筛选目标时 **只返回 `is_online = false` 的设备**——也就是说，与 Gateway 维持 WebSocket 长连接的「在线」设备会被跳过，不会触发任何投递（即使在统计中也不会被计入「目标设备数」）。

WebSocket 长连接当前仅用来维护实时在线态；**面向在线设备的实时消息推送在网关侧尚未实现**，不要依赖此通路下发关键消息。需要确保所有目标都收到的，可在发送前提示用户退出应用以触发离线状态，或选择基于厂商通道的推送策略。
:::

### 推送内容

#### 📝 基本内容

**推送标题**：
- **要求**：必填，1-200个字符
- **作用**：推送通知的主标题
- **建议**：简洁明了，突出重点

**推送内容**：
- **要求**：必填，1-1000个字符
- **作用**：推送通知的详细内容
- **建议**：内容具体，包含行动指引

**角标数量** (`badge`)：
- **作用**：iOS 应用图标上的红色数字（Android 走厂商角标方案，效果以系统为准）
- **格式**：非负整数
- **默认值**：不传时后端落库为 `1`（不会保持原值）；显式传 `0` 可清零

#### 🎯 高级配置

**自定义载荷 (Payload)**：
推送通知可携带自定义数据，用于应用内处理。

**基础载荷字段**：
- **action** - 动作类型（如 `open_page`、`open_url`）
- **url** - 跳转链接（如 `https://example.com`）
- **data** - 自定义数据（JSON 格式字符串）

**基础载荷示例**：
```json
{
  "action": "open_page",
  "url": "https://example.com/news/123",
  "data": "{\"page\":\"news\",\"id\":123}"
}
```

#### 🤖 Android 厂商特定参数

为了优化在不同 Android 厂商设备上的推送效果，DooPush 支持厂商特定的推送参数：

**华为 HMS 参数**：
- `importance` - 消息分类，`NORMAL`（服务与通讯类，默认，不受频控）/ `LOW`（资讯营销类，受频控）
- `category` - 消息分类，`IM`（即时通讯，默认）/`VOIP`/`SUBSCRIPTION` 等，参考华为 Push Kit 自分类

**荣耀推送参数**：
- `importance` - 消息重要性等级（NORMAL/HIGH）
- `ttl` - 消息存活时间（秒）
- `target_user_type` - 目标用户类型（0/1）

**小米推送参数**（参考 `XiaomiMessage` 字段）：
- `pass_through` - `0`=通知消息 / `1`=透传消息
- `notify_type` - 提醒类型位掩码，`1`=声音 / `2`=震动 / `4`=指示灯，可按位组合（如 `7` = 同时开启三种）
- `time_to_live` - 消息存活时间（**毫秒**，注意与 vivo 的「秒」不同）
- `channel_id` - 推送通道 ID（用于指定推送通道类型）

**OPPO推送参数**（参考 `OppoMessage` 字段）：
- `channel_id` - 通知渠道 ID（突破默认通道单设备单日 1 条限制需要的「公信通道」ID）
- `category` - 消息分类，UI 下拉提供 10 项：`IM` / `ACCOUNT` / `DEVICE_REMINDER` / `ORDER` / `TODO` / `SUBSCRIPTION` / `NEWS` / `CONTENT` / `MARKETING` / `SOCIAL`
- `notify_level` - 通知级别（仅 `1` / `2` / `16` 三个合法值）
- `off_line` - 是否离线消息（布尔）
- `off_line_ttl` - 离线消息的存活时间（秒）

**VIVO推送参数**（与 vivo 推送 API 字段一一对应）：
- `classification` - 消息分类，`0`=运营消息 / `1`=系统消息
- `notify_type` - 通知类型，`1`=通知栏 / `2`=透传
- `skip_type` - 跳转类型，`1`=打开应用 / `2`=打开 URL / `3`=自定义
- `skip_content` - 跳转内容（配合 `skip_type` 使用）
- `network_type` - 网络类型，`-1`=不限 / `1`=仅 WiFi
- `time_to_live` - 离线保存时长（秒）

**魅族推送参数**（与魅族 Push API 字段对应；服务端会按字段位置打包到 `noticeBarInfo` / `noticeExpandInfo` / `clickTypeInfo` 等子结构）：
- `notice_msg_type` - 消息分类，`0`=公信 / `1`=私信
- `notice_bar_type` - 通知栏样式，`0`=标准 / `2`=原生
- `notice_expand_type` - 展开方式，`0`=标准 / `1`=文本 / `2`=大图
- `click_type` - 点击动作，`0`=打开应用 / `1`=打开页面（配合 `activity`）/ `2`=打开 URI（配合 `url`）
- `off_line` - 离线消息，`0`=否 / `1`=是
- `valid_time` - 有效时长（小时），`1-72`
- `subtitle` - 通知副标题（VIP 功能）
- `pull_down_top` - 即时置顶，`0`=否 / `1`=是
- `callback` / `callback_param` / `callback_type` - 推送回执相关，`callback_type`：`1`=送达 / `2`=点击 / `3`=送达+点击

**厂商参数示例**：
```json
{
  "action": "open_page",
  "url": "https://example.com/news/123",
  "xiaomi": {
    "pass_through": 0,
    "notify_type": 1,
    "time_to_live": 3600000
  },
  "oppo": {
    "channel_id": "important",
    "category": "SERVICE",
    "notify_level": 2
  },
  "vivo": {
    "classification": 1,
    "notify_type": 1,
    "skip_type": 1,
    "time_to_live": 3600
  },
  "meizu": {
    "notice_msg_type": 0,
    "notice_bar_type": 0,
    "click_type": 0,
    "off_line": 1,
    "valid_time": 24,
    "subtitle": "重要通知"
  }
}
```

### 使用消息模板

页面顶部的「消息模板（可选）」下拉用于快速填充标题与内容：

1. 在「消息模板」下拉中选择目标模板
2. 系统自动应用模板内容到「推送标题」和「推送内容」
3. 如模板包含变量（如 `{{username}}`），先填写变量值再发送

### 发送操作

页面底部按钮文案随「定时发送时间」字段联动：

- **留空时间**：按钮显示「立即发送」，提交后立即创建推送日志并异步投递
- **填写未来时间**：按钮显示「定时发送」，会创建一条定时任务进入「定时推送」列表，到点后由调度器执行

**注意事项**：

- 定时时间必须是未来时间（前端校验）
- 创建后的定时任务可在「定时推送」页面编辑、暂停、恢复或删除
- 立即发送的状态变化通过「推送历史」页面的状态列查看（`pending` → `sent` / `failed`）

## ⏰ 定时推送

定时推送功能允许您预设推送任务，在指定时间自动发送，支持一次性和重复推送。

### 查看定时推送

在左侧导航点击 **"定时推送"** 进入定时推送管理页面。

**列表列**：
- **推送信息** - 一列内同时展示推送标题与内容预览
- **执行时间** - 计划执行的时间
- **重复类型** - 一次性 / 每日 / 每周 / 每月
- **推送类型** - 单设备 / 批量 / 标签 / 广播 / 分组
- **状态** - 等待中 (`pending`) / 运行中 (`running`) / 已完成 (`completed`) / 失败 (`failed`) / 已暂停 (`paused`)
- **创建时间** - 任务创建时间

### 定时推送统计

页面顶部显示定时推送统计：
- **总任务数** - 所有定时推送任务数量
- **等待中** - 等待执行的任务数量
- **运行中** - 正在执行的任务数量
- **已完成** - 执行完成的任务数量
- **失败** - 执行失败的任务数量
- **暂停** - 被暂停的任务数量

### 创建定时推送

**创建步骤**：
1. 点击 **"创建定时推送"** 按钮
2. 填写推送内容（与普通推送相同）
3. 设置执行时间和重复规则
4. 点击 **"创建"** 按钮

**对话框关键字段**：

- **消息模板（可选）** - 与「发送推送」页面一致的模板下拉
- **推送标题** / **推送内容** - 必填
- **动作类型** / **跳转链接** / **额外数据** - 自定义载荷
- **推送类型** - 单设备 / 批量 / 标签 / 广播 / 分组（与立即推送相同）
- **目标配置** / **选择设备分组** - 跟随推送类型展示对应输入
- **执行时间** - 首次执行时间（必须为未来时间）
- **时区** - 默认 `Asia/Shanghai`
- **重复类型** - `none`（一次性）/ `daily`（每日）/ `weekly`（每周）/ `monthly`（每月）
- **重复配置** - 仅在「每周 / 每月」时出现，配置具体哪一天

### 管理定时推送

行末尾「⋮」操作菜单内的动作：

- **立即执行**（仅 `pending` 时可见）：跳过等待，立刻触发一次执行
- **暂停任务**（仅 `running` 时可见）：将任务置为 `paused`，调度器不再到点拉起
- **恢复任务**（仅 `paused` 时可见）：将任务置回 `pending`，下次到点会被调度
- **编辑任务**（任意状态都可打开）：修改推送内容、执行时间、重复规则、状态等
- **删除任务**（任意状态都可执行）：从列表中永久移除

**删除任务**：
- 删除不需要的定时推送任务
- 删除后无法恢复

## 📊 推送历史

推送历史记录了所有推送活动的详细信息和执行结果。

### 查看推送历史

在左侧导航点击 **"推送历史"** 进入推送历史页面。

**列表列**：
- **推送信息** - 一列内同时展示推送标题与内容预览
- **目标** - 目标类型（单设备 / 批量 / 标签筛选 / 广播 / 分组）
- **状态** - `pending`(发送中) / `sent`(成功) / `failed`(失败)
- **成功率** - 成功送达 / 目标设备数 的百分比
- **发送时间** - 推送发送的时间

### 筛选和搜索

**状态筛选**：
- **全部状态** / **已发送** (`sent`) / **发送中** (`processing`) / **失败** (`failed`) / **待发送** (`pending`)

**平台筛选**：
- **全部平台** / **iOS** / **Android**

（按厂商通道筛选未在此页面提供，可在「设备管理」页面按设备 `channel` 字段筛选后再发送针对性推送）

**搜索功能**：
- 顶部搜索框支持按推送标题或内容关键词搜索（实时筛选）

### 查看推送详情

点击列表行打开详情对话框，对话框采用三 Tab 结构：

- **推送概览**
  - 「推送信息」卡片：推送标题、推送状态、推送内容、推送载荷（带复制按钮）
  - 「推送统计」卡片：目标设备、成功发送、发送失败、成功率
- **发送结果** - 按设备列出每条投递的成功 / 失败状态与错误信息
- **数据分析** - 该条推送的送达 / 点击 / 打开等行为统计

列表中提供 **「复用推送」** 操作：基于该条历史推送一键回填到「发送推送」表单，便于二次编辑后再发送。

## 📈 推送统计

推送统计提供详细的推送效果分析和数据洞察。

### 查看推送统计

在左侧导航点击 **"推送统计"** 进入推送统计页面，页面顶部可以切换时间范围（7 天 / 30 天 / 90 天）。

### 概览卡片（4 张）

| 卡片 | 含义 |
|------|------|
| **推送成功率** | `success_pushes / total_pushes`，下方副标显示分子分母 |
| **点击率** | 用户点击推送通知的比率 |
| **打开率** | 用户从推送打开应用的比率 |
| **活跃设备** | 时间窗口内可接收推送的设备数 |

### 详细图表（4 个 Tab）

- **推送趋势** - 折线图展示所选时间窗口内每日的推送总量、成功量、失败量
- **平台分布** - iOS / Android 在 success_count、failed_count 等维度上的对比
- **厂商分析** - Android 各厂商通道（FCM / 华为 / 荣耀 / 小米 / OPPO / VIVO / 魅族）的成功率与占比
- **用户参与** - 点击 / 打开等互动行为分析

## 📝 消息模板

消息模板功能帮助您创建可复用的推送内容，提高推送效率和内容一致性。

### 查看消息模板

在左侧导航点击 **"消息模板"** 进入模板管理页面。

**显示信息**：
- **模板名称** - 模板的识别名称
- **推送标题** - 模板的推送标题
- **推送内容** - 模板的推送内容（截取显示）
- **支持平台** - iOS、Android、全部
- **语言** - 模板的语言版本
- **状态** - 启用、禁用
- **创建时间** - 模板创建日期

### 创建消息模板

**创建步骤**：
1. 点击 **"创建模板"** 按钮
2. 填写模板基本信息
3. 设置模板变量（可选）
4. 点击 **"创建"** 按钮

**基本信息**：

#### 📝 模板名称
- **要求**：必填，1-100个字符
- **作用**：模板的识别名称
- **建议**：使用有意义的命名，如 `会员升级通知`、`订单确认消息`

#### 📱 支持平台
- **全平台** (`all`)
- **仅 iOS** (`ios`)
- **仅 Android** (`android`)

#### 🌐 语言设置

下拉选项：

- **简体中文** (`zh-CN`)，默认值
- **繁体中文** (`zh-TW`)
- **English** (`en-US`)
- **日本語** (`ja-JP`)
- **한국어** (`ko-KR`)

#### 📄 推送内容
- **推送标题** - 模板的标题内容
- **推送内容** - 模板的详细内容（在内容中使用 `{{变量名}}` 占位即可被自动识别为变量）

### 模板变量

**变量语法**：

- 使用双大括号包围变量名：`{{变量名}}`
- 变量名只能包含字母、数字、下划线
- 在编辑「推送内容」时，文本中出现的 `{{xxx}}` 会自动加入到下方变量列表

**变量配置（每条变量四个字段）**：

| 字段 | 说明 |
|------|------|
| 变量名 | 自动来自模板内容中的 `{{变量名}}` |
| 类型 | `string` / `number` / `boolean` |
| 描述 | 变量用途说明 |
| 默认值 | 调用方未传值时的回退（可选） |

**示例**：

```
尊敬的 {{username}}，您的订单 {{order_id}} 已确认。
```

### 使用模板

在「发送推送」页面顶部「消息模板（可选）」下拉中选择目标模板，系统会把模板的标题与内容自动填入推送表单；如有变量，先填变量值，提交时由后端用 `{{name}}` 占位替换。

### 模板管理

- **编辑** - 修改名称 / 平台 / 语言 / 标题 / 内容 / 变量
- **预览** - 在「预览模板」对话框查看应用变量后的最终展示效果
- **渲染调试** - 在「渲染模板」对话框输入变量值即时查看渲染输出
- **启用 / 禁用** - 控制模板能否在「发送推送」下拉中被选中
- **删除** - 永久移除模板

## 💡 推送最佳实践

### 内容优化

**标题建议**：
- 控制在 20-30 个字符内
- 突出核心信息和价值
- 使用动作词汇吸引点击

**内容建议**：
- 提供具体的行动指引
- 包含时间敏感性信息
- 个性化内容提高相关性

### 时间优化

**推送时机**：
- 基于用户活跃时间发送
- 避开用户休息时间
- 考虑用户所在时区

**频率控制**：
- 避免过度推送影响体验
- 根据用户偏好调整频率
- 重要消息优先级处理

### 目标优化

**精准定位**：
- 利用标签和分组功能
- 基于用户行为数据推送
- A/B 测试优化效果

**效果监控**：
- 持续监控推送数据
- 根据统计结果调整策略
- 优化推送时间和内容

## ❓ 常见问题

### Q: 推送失败的常见原因？
A: 主要原因包括设备 Token 失效、推送配置错误、设备离线、用户关闭推送权限等。

### Q: 如何提高推送点击率？
A: 优化推送内容、选择合适推送时间、精准用户定位、提供有价值的信息。

### Q: 定时推送可以取消吗？
A: 可以。未执行的定时推送可以编辑、暂停或删除。

### Q: 模板变量有数量限制吗？
A: 建议单个模板的变量数量不超过 10 个，确保内容清晰易维护。

### Q: 广播推送会影响所有用户吗？
A: 广播推送实际只覆盖应用下满足平台筛选且 `is_online = false` 的设备（与「在线设备目前会被跳过」的说明一致）；在线设备由 Gateway 维持长连接但当前不会收到这条广播。建议谨慎使用，确保内容对所有用户都有价值。
