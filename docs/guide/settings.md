# 系统设置

系统设置提供了平台的配置管理、安全审计和个人偏好设置功能。通过系统设置，您可以配置推送服务、监控操作日志、管理个人账号信息。

## ⚙️ 推送配置

推送配置是 DooPush 正常工作的基础，需要为不同平台配置相应的推送服务参数。

### 查看推送配置

在左侧导航点击 **"推送配置"** 进入配置管理页面。

**配置分类**：
- **应用配置** - 应用基本配置信息
- **iOS 配置** - iOS APNs 推送配置
- **Android 配置** - Android 多厂商推送配置（FCM / 华为 / 荣耀 / 小米 / OPPO / vivo / 魅族）

### 应用配置

**基本信息**：
- **应用名称** - 应用的显示名称
- **包名** - 应用的唯一标识符
- **支持平台** - iOS、Android 或两者
- **应用状态** - 启用或禁用状态

### iOS 推送配置

iOS 平台使用 Apple Push Notification service (APNs) 进行推送。

#### 创建 iOS 配置

**创建步骤**：
1. 点击 **"添加配置"** 按钮
2. 选择平台为 **"iOS"**
3. 选择通道为 **"APNs"**
4. 填写 APNs 配置信息
5. 点击 **"保存"** 按钮

**配置参数**：

#### 🔑 Key ID
- **作用**：Apple 开发者账号中的推送密钥 ID
- **获取方式**：在 Apple 开发者中心的 "Certificates, Identifiers & Profiles" > "Keys" 中查看
- **格式**：10 位字符串，如 `ABC1234DEF`

#### 👥 Team ID
- **作用**：Apple 开发者团队 ID
- **获取方式**：在 Apple 开发者中心的 "Membership" 页面查看
- **格式**：10 位字符串，如 `DEF5678GHI`

#### 📱 Bundle ID
- **作用**：iOS 应用的 Bundle Identifier
- **要求**：必须与应用的实际 Bundle ID 保持一致
- **格式**：域名反向格式，如 `com.yourcompany.yourapp`

#### 🔐 Private Key
- **作用**：APNs 推送私钥内容
- **格式**：完整的 .p8 私钥文件内容，包含 BEGIN 和 END 标记
- **安全**：私钥信息将加密存储

**私钥格式示例**：
```
-----BEGIN PRIVATE KEY-----
MIGTAgEAMBMGByqGSM49AgEGCCqGSM49AwEHBHkwdwIBAQQg...
-----END PRIVATE KEY-----
```

#### 🌍 推送环境
- **开发环境** - 用于开发和测试阶段
- **生产环境** - 用于正式发布的应用

::: warning ⚠️ 环境选择
确保选择正确的推送环境，否则推送将无法送达。开发版本应用使用开发环境，App Store 版本使用生产环境。
:::

### Android 推送配置

Android 平台支持多个推送通道，包括 FCM 和各厂商通道。

::: tip 📱 Android SDK 状态
Android SDK 已完成开发，支持全部厂商推送通道，可正常配置和使用。
:::

**支持的推送通道**：
- **FCM** - Firebase Cloud Messaging
- **华为** - HMS Push Kit
- **荣耀** - 荣耀推送
- **小米** - 小米推送
- **OPPO** - ColorOS 推送
- **VIVO** - vivo 推送
- **魅族** - Flyme 推送

### 消息回执

DooPush 暴露 `POST /api/v1/apps/callback/{vendor}` 接收厂商推送回执（无需认证），按 `(应用, 厂商, 日期)` 累加到回执统计表。各厂商配置位置不同：

#### 在 DooPush 配置弹窗里填写（每条消息携带）

OPPO / 小米 / 魅族 / vivo 的协议要求每条推送在 payload 里带回执地址。在创建 / 编辑配置弹窗的「消息回执」字段填入：

| 通道 | 字段含义 | 应填内容 |
|------|----------|----------|
| **OPPO** | 回执 URL | `https://<your-doopush-host>/api/v1/apps/callback/oppo` |
| **小米** | 回执 URL | `https://<your-doopush-host>/api/v1/apps/callback/xiaomi` |
| **魅族** | 回执 URL | `https://<your-doopush-host>/api/v1/apps/callback/meizu` |
| **vivo** | 回执 ID（不是 URL） | vivo 推送后台分配的 callback id；DooPush 会自动以 `callback.param=vivo` 配套发送 |

#### 在厂商后台配置（一次性，不在 DooPush 弹窗）

华为 HMS、荣耀的回执是在厂商开发者后台一次性注册回调地址，**不需要**也**不应该**在 DooPush 配置弹窗里填。

| 通道 | 配置位置 | 应填内容 |
|------|----------|----------|
| **华为** | 华为开发者联盟 → Push Kit → 配置消息回执 | `https://<your-doopush-host>/api/v1/apps/callback/huawei` |
| **荣耀** | 荣耀开发者平台 → 管理中心 → 推送服务 → 应用回执 | `https://<your-doopush-host>/api/v1/apps/callback/honor` |

DooPush 这一侧的接收路由已经实现，配好厂商后台后回执会自动累加到统计表。

::: tip 💡 荣耀通道的 BadgeClass
荣耀通道在配置弹窗里有「应用入口Activity类全路径」字段，这是用于角标设置的 Activity 类名（如 `com.example.app.MainActivity`），不是回执地址，请按字面填写。
:::

#### 暂不支持的通道

| 通道 | 原因 |
|------|------|
| **FCM** | FCM 的回执通过 `delivery_receipt_requested` + Google Cloud Pub/Sub 订阅传递，而非 webhook URL，DooPush 暂未实现该流程 |
| **iOS APNs** | Apple APNs 协议不提供 delivery webhook，仅可通过 Apple 推送通知控制台 / Metrics 查看状态 |

完整接口字段见 [API 文档 - 消息回执](/api/push-apis.md#消息回执接口)。

### 配置管理

#### 编辑配置
1. 在配置列表中找到目标配置
2. 点击 **"编辑"** 按钮
3. 修改配置参数
4. 点击 **"保存"** 按钮

#### 测试配置
1. 在配置列表点击 **"测试"** 按钮
2. 在「测试推送配置」对话框中填写：
   - **测试设备ID** - 已注册到该应用的设备 ID 或 Token
   - **测试标题** - 默认为 `推送配置测试`
   - **测试内容** - 默认为 `这是一条配置测试消息，用于验证推送通道是否正常工作`
3. 点击发送，对话框顶部会展示当前配置（平台 / 通道 / 环境）以便确认
4. 后端调用 `POST /apps/{appId}/config/test` 发送一条真实推送，结果显示在对话框底部

#### 删除配置
1. 点击配置的 **"删除"** 按钮
2. 确认删除操作

::: danger 🔒 删除警告
删除推送配置后，相关推送功能将停止工作，请谨慎操作。
:::

### 配置状态

**状态说明**：
- 🟢 **正常** - 配置有效，推送服务正常
- 🔴 **异常** - 配置错误或服务异常
- ⏸️ **禁用** - 配置被手动禁用

**常见问题**：
- **证书过期** - 需要更新 iOS 推送证书
- **密钥错误** - 检查 Key ID、Team ID 是否正确
- **Bundle ID 不匹配** - 确保与应用实际 Bundle ID 一致

## 📋 审计日志

审计日志记录了平台上的所有重要操作，用于安全监控和问题追溯。

### 查看审计日志

在左侧导航点击 **"审计日志"** 进入日志查看页面。

**日志信息**：
- **操作时间** (`created_at`)
- **操作用户** (`user_name`，未冗余时退化为 `用户#<id>`)
- **操作类型** (`action`，带中文标签)
- **操作资源** (`resource`，带中文标签 + `resource_id`)
- **IP 地址** (`ip_address`) 与 **User-Agent** (`user_agent`)
- **操作详情** (`details` JSON)
- **变更前 / 变更后** (`before_data` / `after_data`，仅有变更才填)

### 操作类型

**常见操作类型**：
- **创建** (`create`) - 创建新的资源
- **更新** (`update`) - 修改现有资源
- **删除** (`delete`) - 删除资源
- **推送** (`push`) - 发送推送
- **登录** (`login`) - 用户登录系统
- **登出** (`logout`) - 用户登出系统

### 资源类型

**记录的资源类型**：
- **应用** (App) - 应用管理操作
- **设备** (Device) - 设备管理操作
- **推送** (Push) - 推送发送操作
- **配置** (Config) - 推送配置操作
- **模板** (Template) - 消息模板操作
- **分组** (Group) - 设备分组操作
- **API密钥** (API Key) - API 密钥管理
- **定时推送** (Scheduled Push) - 定时推送管理

### 筛选和搜索

#### 高级筛选

**操作筛选**：
- 按操作类型筛选（创建、更新、删除等）
- 按资源类型筛选（应用、设备、推送等）

**时间筛选**：
- **开始时间** - 筛选的起始时间
- **结束时间** - 筛选的结束时间
- 支持自定义时间范围

**用户筛选**：
- **用户名** - 按执行操作的用户筛选
- **IP 地址** - 按操作来源 IP 筛选

#### 快速搜索
- 在搜索框中输入关键词
- 支持搜索操作内容和资源信息

### 查看详情

**日志详情信息**：
- **基本信息** - 时间、用户、IP / User-Agent、操作类型与资源
- **操作详情** - `details` JSON 中的具体描述
- **变更前后** - 若操作涉及更新，会同时展示 `before_data` 与 `after_data`

### 导出日志

**导出功能**：
1. 设置筛选条件
2. 点击 **"导出"** 按钮
3. 后端生成 Excel (`.xlsx`) 文件并直接下载

**导出内容**：当前筛选结果对应的全部日志记录，包含完整的操作信息和上下文。

### 统计分析

切换到「操作统计」页签查看（卡片字段以实际页面为准）：

- **总操作数** - 时间窗口内的全部操作计数
- **操作类型** - 按 `(action, resource)` 聚合的操作分布
- **活跃用户** - 时间窗口内有操作记录的用户数
- **平均操作** - 单用户平均操作次数

### 权限控制

权限粒度是 **「用户 × 应用」**，由 `UserAppPermission` 表保存，没有全局超级管理员角色：

- **owner**（应用所有者）：对该应用的全部接口都有权限
- **developer**（开发者）：可创建 / 编辑应用资源（设备、推送、配置、模板等）
- **viewer**（只读）：可查看包括审计日志在内的所有数据，但不能修改

只有对该应用持有任意角色的用户才能看见对应应用的审计日志；其它应用的日志对该用户不可见（包括 owner，无法跨应用查看）。

## 👤 个人设置

个人设置允许您管理账号信息、安全设置和界面偏好。

### 访问个人设置

点击右上角的用户头像，选择 **"个人设置"**，或在左侧导航中点击相关设置项。

### 个人资料

#### 查看和编辑资料

**字段**：
- **用户名**（disabled）- 登录用户名，不可修改
- **邮箱地址**（disabled）- 注册邮箱，不可修改
- **显示昵称** - 系统中显示的名称，最多 50 个字符
- **头像** - 拖拽或点击「选择头像」上传，支持 JPEG / PNG / GIF / WebP / SVG，单文件最大 5MB

**保存**：填写完成后点击 **"更新资料"** 按钮提交。

### 账号安全

#### 修改密码

通过左侧导航 **「设置 → 账户」** 进入修改密码表单：

1. 在「原密码」输入当前登录密码
2. 在「新密码」输入新密码（至少 6 个字符）
3. 在「确认新密码」再次输入新密码
4. 点击 **"修改密码"** 按钮提交

### 外观设置

进入「设置 → 外观」可调整：

#### 主题

- **浅色** - 适合白天使用
- **深色** - 适合夜晚使用
- **跟随系统** - 根据系统外观自动切换

#### 字体

控制台界面字体样式可在外观设置中切换。

::: tip 跨账号生效范围
主题保存在浏览器 Cookie，字体保存在浏览器本地，更换设备 / 浏览器需重新设置。
:::

## 🔒 安全建议

### 账号安全

**密码管理**：
- 定期更换密码（建议3-6个月）
- 使用强密码，避免使用生日、姓名等
- 不要在多个平台使用相同密码

**登录安全**：
- 退出时及时登出账号
- 不要在公共设备上保存登录状态
- 发现异常登录立即修改密码

### 推送配置安全

**证书管理**：
- 推送证书和私钥妥善保管
- 不要在代码中硬编码证书信息
- 定期检查证书有效期

**访问控制**：
- 限制推送配置的访问权限
- 定期审查配置人员权限
- 重要操作启用审批流程

### 日志监控

**定期检查**：
- 定期查看审计日志
- 关注异常操作和登录
- 发现问题及时处理

**告警设置**：
- 设置重要操作的通知
- 监控批量操作和删除行为
- 建立安全事件响应流程

## ❓ 常见问题

### Q: iOS 推送配置后测试失败怎么办？
A: 检查 Key ID、Team ID、Bundle ID 是否正确，确认私钥格式完整，验证推送环境选择。

### Q: 审计日志可以保存多长时间？
A: 系统默认保留 365 天的审计日志（由 `AUDIT_RETENTION_DAYS` 环境变量控制，部署时可调整），超过期限的日志会被自动清理任务（默认每 24 小时执行一次）批量删除。

### Q: 忘记密码怎么办？
A: 平台当前的「忘记密码」流程仍在开发中，登录页提供入口但点击后不会真的发送重置邮件。如确实遗忘，请联系管理员在数据库直接重置 `users.password` 字段。

### Q: 可以同时配置多个 iOS 证书吗？
A: 可以。支持为同一应用配置多个推送配置，用于不同环境或备份。

### Q: 如何确保账号安全？
A: 使用强密码、定期修改密码、及时登出、监控登录记录。
